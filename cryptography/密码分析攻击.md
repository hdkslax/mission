# 密码分析攻击

### 唯密文攻击 ciphertext-only attack

通过一些用相同算法加密的密文，恢复尽可能多的明文，最好能推算出加密消息的密钥，或者找出一个算法从密文推导出明文。

已知：$C_1 = E_k(P_1), ..., C_i = E_k(P_i)$

推出：$P_1, P_2, ..., P_i$, $K$, 或算法$C_{i+1}=E_k(P_{i+1})$ 

### 已知明文攻击 known-plaintext attack

已知一些明文和对应的密文，推导出加密密钥或一个算法（可以对用相同密钥加密的任何新消息进行解密）。

已知：$P_1, C_1 = E_k(P_1), ..., P_i, C_i = E_k(P_i)$

推出：$K$, 或算法$C_{i+1}=E_k(P_{i+1})$ 

### 选择明文攻击 chosen-plaintext attack

已经一些明文和对应密文，攻击者还能选择特定的明文块进行加密，那些块可能产生更多关于密钥的信息。根据这些，推导出密钥或一个算法，此算法可以对用相同密钥加密的任何新消息进行解密。

已知：$P_1, ..., P_i, C_1=E_k(P_1), ..., C_i=E_k(P_i)$, 其中$P_1, ..., P_i$可由密码分析者选择

推出：$K$, 或算法$C_{i+1}=E_k(P_{i+1})$

### 自适应选择明文攻击 adaptive-chosen-plaintext attack

选择明文攻击的特殊情况。

密码分析者不禁能选择被加密的明文，还能基于以前加密的结果修正这个选择。

攻击者可选择较小的明文块，然后基于第一块的结果选择第二个明文块，以此类推。

### 选择密文攻击 chosen-ciphertext attack

这种攻击主要用于公开密钥算法。

攻击者可选择不同的被加密的密文，并得到对应的明文。然后推出密钥。

已知：$C_1, ... C_i, P_1=D_k(C_1), ..., P_i=D_k(C_i)$

推出：$K$

### 选择文本攻击 chosen-text attack

选择文本攻击 = 选择明文攻击 + 选择密文攻击

### 选择密钥攻击 chosen-key attack

密码分析者具有不同密钥之间关系的有关知识。

### 软磨硬泡攻击 rubber-hose cryptanalysis

密码分析者威胁、勒索，或者折磨某人，知道他给出密钥为止。

行贿有时称为**购买密钥攻击 (purchase-key attack)**。

这些都是非常有效的攻击，并且经常是破译算法的最好途径。

# 其他攻击方式

### 中间人攻击 man-in-the-middle attack

- A将公钥传给B，却被M截取，M将这个公钥替换成自己的传给B
- B将公钥传给A，仍被M截取，M将这个公钥替换成自己的传给A

### 字典式攻击 dictionary attack

- 用单项散列函数对所有口令进行运算，并将结果储存。
- 截获到加密口令文件时，将它与自己可能的口令文件进行比较，观察哪个更匹配。
